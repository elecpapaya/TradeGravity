TradeGravity is recruiting three people outside the implementation team for the task in [USER_TESTING.md](USER_TESTING.md). ## Maintainer checklist

Before the session:

- confirm the participant is outside the implementation team and can use the public site;
- give only the task prompt in [USER_TESTING.md](USER_TESTING.md), not a walkthrough;
- state that participation is voluntary and may stop at any time;
- ask separately whether a non-identifying summary may be published;
- record the tested release, browser, and viewport class.

After the session:

- create a **User-study feedback** issue only when publication consent was given;
- use an alias such as `P1` and omit identity, contact details, employer, recording, credentials, and sensitive data;
- classify the highest severity and link each actionable finding to a follow-up issue;
- update the summary table in [USER_TESTING.md](USER_TESTING.md) only from those real public records.

No gift, employment opportunity, product access, or application outcome is promised in exchange for participation.
